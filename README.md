## О проекте

Этот проект исследует **проблему неконтролируемой генерации LLM** и демонстрирует, как техники **Output Manipulation** позволяют получить предсказуемые, структурированные, верифицируемые ответы.

На реальном датасете книжных отзывов реализованы и сравниваются три стратегии промптинга:

| Стратегия | Описание |
|-----------|----------|
| `baseline` | Свободный промпт — демонстрирует нестабильность |
| `structured` | JSON-схема + список допустимых меток |
| `few_shot` | 2 реальных примера + строгая схема |

**Результат:** structured и few_shot дают 100% успешный JSON-вывод, baseline — 0%.

---

## Проблема и мотивация

Большие языковые модели (LLM) по умолчанию генерируют свободный текст — без гарантий формата, без контроля над используемой терминологией. Это создаёт критические проблемы при интеграции LLM в промышленные системы:

- **Непредсказуемый формат ответа** — парсинг ломается при изменении стиля модели
- **Произвольные метки** — модель придумывает собственную классификацию вместо использования заданной
- **Отсутствие верификации** — нельзя автоматически проверить корректность вывода

Данный проект решает эти проблемы через комплекс техник Output Manipulation.

## Архитектура решения

```
Входной отзыв
      │
      ▼
┌─────────────────┐
│  Стратегия      │  baseline / structured / few_shot
│  промптинга     │
└────────┬────────┘
         │ промпт
         ▼
┌─────────────────┐
│   LLM API       │  OpenRouter
│  (call_llm)     │
└────────┬────────┘
         │ raw text
         ▼
┌─────────────────┐
│  parse_json_    │  3 попытки извлечения JSON
│  response()     │  (прямой / ```json``` / regex)
└────────┬────────┘
         │ dict или None
         ▼
┌─────────────────┐
│  validate_      │  Guardrails:
│  spans()        │  - проверка меток по whitelist
└────────┬────────┘  - fuzzy match при ошибке
         │           - fallback на "Анализ"
         ▼           - проверка sentiment
     Результат
  {spans, errors, fixed}
```

## Три стратегии промптинга

### 1. Baseline (контрольная группа)
```
Раздели следующий отзыв на смысловые части и определи тему каждой части.

Отзыв: <text>
```
Никаких ограничений на формат → модель отвечает произвольным текстом → JSON не парсится.

### 2. Structured Output
```
Раздели отзыв на смысловые спаны. Для каждого спана определи метку.

ДОПУСТИМЫЕ МЕТКИ (только из этого списка):
- Анализ
- Аудитория
- Автор
- ...

ФОРМАТ ОТВЕТА — строго JSON, без лишнего текста:
{"spans": [{"text": "...", "label": "...", "sentiment": "positive|negative|neutral"}]}
```
Явная JSON-схема + whitelist меток → 100% успешный парсинг.

### 3. Few-Shot
2 реальных аннотированных примера из датасета + та же схема → модель «видит» ожидаемый паттерн.

## Guardrails и валидация

Модуль `validate_spans()` реализует многоуровневую защиту:

```python
VALID_LABELS = {
    "Эмоции", "Вывод", "Анализ", "Читательские ожидания",
    "Сюжет", "Герои", "Экспозиция", "Нарратив", "Стиль",
    "Миры", "Сравнение", "Автор", "Жанр", "Контекст",
    "Аудитория", "Мораль", "Интертекст", "Озвучка",
    "Цитаты", "Спойлер", "Оформление", "Перевод"
}
```

**Уровни защиты:**

1. **JSON-парсинг с fallback** — три независимые попытки извлечения JSON
2. **Whitelist меток** — любая метка вне множества `VALID_LABELS` отклоняется
3. **Fuzzy matching** — нечёткий поиск для исправления опечаток (`"Анализ книги"` → `"Анализ"`)
4. **Fallback-метка** — если fuzzy match не сработал, ставится `"Анализ"`
5. **Sentiment guardrail** — допускаются только `positive / negative / neutral`
6. **Логирование всех исправлений** — поля `errors`, `fixed` в результате

## Анализ и выводы

### Почему baseline всегда проваливается?
Модель без явных инструкций выбирает наиболее «естественный» формат ответа — повествовательный текст. JSON не является стандартным форматом человеческого общения, поэтому без явного указания модель к нему не прибегает.

### Почему structured работает на 100%?
Три механизма совместно обеспечивают результат:
- **Явная схема** устраняет неопределённость формата
- **System prompt** задаёт роль эксперта
- **Whitelist** сужает пространство допустимых ответов

### Почему few_shot даёт меньше спанов?
Few-shot примеры содержат 2-3 спана → модель калибрует свой вывод под длину примеров. Это известный эффект длинного контекста.

### Дальнейшие направления
- **Chain-of-Thought + structured** — рассуждение перед финальным JSON
- **Self-consistency** — несколько запросов → голосование по меткам
- **Fine-tuning** на размеченном датасете отзывов
- **Оценка качества** через сравнение с human-аннотацией (F1 по меткам)

## Практическая значимость

Реализованный модуль применим в реальных задачах:

- **Аналитика отзывов** (книги, фильмы, товары) — автоматическая разметка аспектов
- **Контент-модерация** — гарантированный формат ответа для downstream-систем
- **Обучающие датасеты** — автоматическая аннотация корпусов
- **RAG-системы** — структурированное извлечение фактов из текста

Модуль использует OpenRouter — единый шлюз с бесплатным доступом к мощным моделям.

## Быстрый старт

### Требования
- Python 3.9+
- Интернет-соединение
- API-ключ одного из провайдеров (см. ниже)

### Установка
```bash
git clone https://github.com/yfaizrahmanova/tguai-segmenter.git
cd tguai-segmenter
```
*Внешних зависимостей нет — только стандартная библиотека Python.*

### Настройка API

Откройте `segmenter_module.py` и укажите провайдера и ключ:

```python
PROVIDER = "openrouter"
API_KEY  = "ваш_ключ_здесь"
```

| Провайдер | Где получить ключ | Стоимость |
|-----------|-----------------|-----------|
| **OpenRouter** | openrouter.ai → Keys → Create Key | Бесплатный tier |

### Запуск
```bash
python3 segmenter_module.py путь/к/датасету.json
```

### Пример использования в коде
```python
from segmenter_module import segment_review

result = segment_review(
    text="Книга захватила с первых страниц. Герои живые и настоящие.",
    strategy="structured"  # "baseline" | "structured" | "few_shot"
)

print(f"Спанов: {result['n_spans']}")
print(f"Ошибок: {result['n_errors']}")
for span in result['validation']['spans']:
    print(f"  [{span['label']}] {span['text']}")
```

### Запуск полного эксперимента
```python
from segmenter_module import run_experiment, summarize_results

results = run_experiment("merged_1209.json", n_samples=10)
summary = summarize_results(results)

for strategy, stats in summary.items():
    print(f"{strategy}: успех {stats['success_rate']}%")
```
